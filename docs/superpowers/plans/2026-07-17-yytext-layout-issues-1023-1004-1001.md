# YYText Layout Issues #1023, #1004, and #1001 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> `superpowers:subagent-driven-development` (recommended) or
> `superpowers:executing-plans` to implement this plan task-by-task. Use
> `superpowers:test-driven-development` for every production change and
> `superpowers:verification-before-completion` before reporting success.

**Goal:** Correct composed-character caret selection, restore multi-container
layout results with unambiguous Swift imports, and include every same-row line
fragment in `firstRectForRange:`.

**Architecture:** Make three narrow corrections in `YYTextLayout`: choose the
nearest valid composed-character edge, append layouts as multi-container flow
advances, and iterate to the calculated end line while retaining the existing
row guard. Preserve Objective-C selectors and exercise behavior through public
APIs. Add `NS_SWIFT_NAME` only at the header boundary.

**Tech Stack:** Objective-C, Swift 5, UIKit, CoreText, XCTest, Xcode project
files, iOS Simulator.

## Global Constraints

- Follow the approved design in
  `docs/superpowers/specs/2026-07-17-yytext-layout-issues-1023-1004-1001-design.md`.
- Keep all existing Objective-C selectors and failure behavior unchanged.

  selection geometry outside the three confirmed defects.
- Keep the framework deployment target at iOS 8.0 and add no dependency.
- Select an available iOS Simulator dynamically during execution. Do not add a
  simulator name or UDID to repository files.
- Preserve unrelated user changes if the main worktree becomes dirty.
- Produce one implementation commit per GitHub issue.

**Behavior under test:**

- `-[YYTextLayout closestPositionToPoint:]` returns the beginning of a
  composed sequence for a point nearer its leading edge.
- It returns the end for a point nearer its trailing edge.
- The behavior holds for horizontal and vertical containers.
- No returned offset lies inside the UTF-16 range of the sequence.

- [ ] **Step 1: Add the caret regression test source to the test target**

Create `Tests/YYTextLayoutCaretTests.m` and register it in the Tests group and
`YYTextTests` Sources build phase in
`Framework/YYText.xcodeproj/project.pbxproj`.

Use a string with plain characters on both sides of a composed emoji. Build a
layout with a deterministic font and no truncation. Determine the composed
range with `rangeOfComposedCharacterSequenceAtIndex:`. Obtain its leading and
trailing geometry with `caretRectForPosition:` and query points on both halves
with `closestPositionToPoint:`. Repeat using a container whose
`verticalForm = YES`.

Assertions must first prove the fixture is valid: the range length is greater
than one UTF-16 unit, both caret rectangles are non-null, and the two edge
coordinates differ. Then assert the exact start/end offsets and explicitly
assert that neither result is strictly inside the composed range.

### Task 3: Include all same-row fragments in `firstRectForRange:` (#1001)

**Files:**

- Create: `Tests/YYTextLayoutSelectionRectTests.m`
- Modify: `Framework/YYText.xcodeproj/project.pbxproj`
- Modify: `YYText/Component/YYTextLayout.m:1958-2022`

**Behavior under test:**

- When an exclusion path splits a visual row, a range spanning two fragments
  returns a first rectangle covering both fragments.
- A range within one ordinary fragment retains its precise partial rectangle.
- Fragments from later rows are not included.

- [ ] **Step 1: Build a deterministic fragmented-row fixture**

Create `Tests/YYTextLayoutSelectionRectTests.m` and add it to the test target.
Create a wide horizontal container with zero insets and a rectangular
exclusion path through the middle of its first text row. Use a fixed-size font
and enough text for CoreText to create fragments on both sides and at least one
later row.

Locate two consecutive `layout.lines` entries with the same `row`. Assert that
the fixture found such a pair, that their ranges do not overlap, and that a
later line has a different row. This prevents a false pass if CoreText layout
changes.

- [ ] **Step 2: Add the failing fragmented-row assertion and a baseline**

Construct a `YYTextRange` whose start is the first fragment's range start and
whose end is the second fragment's range end with backward affinity. The
expected rectangle is `CGRectUnion(first.bounds, second.bounds)`. Compare all
four components with a small floating-point tolerance.

Add a separate ordinary-container test selecting part of one line. Its
`firstRectForRange:` result must be non-null, have the line height, and begin
and end at the corresponding caret offsets.
